Журнал

Все перемещения товаров на складах фиксируются в журнале перемещений. Однако функционал журнала на этом не ограничивается, он также позволяет удалять действия в масштабе транзакций, возвращать продукты на склад, а также формировать документы в контексте действий.


Чтобы воспользоваться журналом перемещений активируйте меню OneBox наведением курсора, затем кликните левой кнопкой мышки на пункт «Склады» → «Журнал».




Все действия в журнале представлены в виде таблицы:




Таблица позволяет сортировать данные. Происходит сортировка по нажатию на заголовок колонки. Чтобы выполнить сортировку в обратном порядке нужно повторно нажать на заголовок.
Под таблицей находится функционал для экспорта документа в форматы: CSV, XLS, XML. Это реализовано с помощью меню, которое активируется по клику мышкой.
Здесь же находится пагинатор, который позволяет листать заказы постранично. Для этого следует выбрать номер страницы или воспользоваться кнопками «Назад» и «Вперед» для перехода на предыдущую/следующую страницу соответственно.


При наведении курсора на заголовок таблицы изображение курсора изменится на знак «?». Это означает, что в этот момент пользователь может перейти к настройкам отображения таблицы - для этого нажмите правую кнопку мышки.


Страница настроек состоит из нескольких блоков и выглядит следующим образом:




В правом верхнем углу находится знак «Х», который служит для возврата на предыдущую страницу без сохранения изменений.
Первый блок позволяется регулировать отображение колонок. Он представлен множеством checkbox`ов, каждый их которых отвечает за свою колонку в таблице. Если checkbox отмечен, значит колонка отображается, если нет — колонка в таблице отображаться не будет. Важно! Это работает не для всех колонок, некоторые ключевые колонки должны быть видимыми всегда: их checkbox`ы не реагируют на клики.


Следующий блок представлен полями:
«Показывать записей на странице» - отображение количество записей на странице. Максимально допустимое значение - 100 записей.
«Сортировка по умолчанию» - выбор колонки, по которой будет происходить сортировка, также возможность сортировать по порядку или в обратном алфавитном порядке.


Внизу расположены кнопки, которые позволяют завершить работу с формой:
«Сохранить» - нажмите, чтобы изменения вступили в силу.
«Отмена» - возврат на предыдущую страницу без сохранения изменений.


Таблица показывает все типы перемещений, дату перемещения и оператора, который ответственный за процесс перемещения. Также в ней видно, с какого и на какой склад произошло перемещение товара, в каком количестве и по какой цене.
Некоторые поля являются ссылками. Колонка «Оператор» содержит имена, по нажатию на которые система выполнит переход к карточке контакта. Подробней о работе с контактами можно прочесть в разделе «Контакты» или по этой ссылке .

Все поля панели являются критериями для фильтрации и активируются путем нажатия на них левой кнопкой мышки. Для того, чтобы изменения вступили в силу следует нажать на кнопку «Фильтровать», которая появится чуть правей панели.




Если в результате фильтрации список записей не был сформирован — это значит, что записей, соответствующих заданным критериям, в базе не существует.


Выпадающий список «Показывать» позволяет сортировать по типу перемещения. Поля «Дата с» и «Дата по» дают возможность задать временный промежуток для данных в таблице.
Поля «Со склада» и «На склад» - задают фильтрацию по складам.


Колонка «Подробнее» позволяет просмотреть более детальную информацию по каждому перемещению. Независимо от типа перемещения страница с информацией выглядит практически идентично. Пример страницы с оприходованием товаров:




Информация о перемещении для удобства разделена на блоки.

  • Самый верхний блок отображает:
  • дату проведения перемещения
  • откуда и куда был перемещен товар
  • юридическое лицо - используется для формирования документов
  • оператора — ответственный за перемещение товаров
  • сумму — необходимо при продаже товаров

Также в блоке находятся кнопки «Вернуть продукт» (присутствует не всегда, например, при производстве товаров кнопка отображаться не будет) и «Удалить».
Удаление — дает возможность удалить все перемещения продукта из системы. Удаление происходит в масштабе транзакций, это значит, что продукт будет удален со складов, как будто его никогда не было в системе.
Возврат продукта - возвращает продукт на склад, создавая при этом новый этап его перемещений в журнале. В названии этапа будет отмечено, что состоялся возврат. Однако такой этап не является завершающим, система позволяет выполнить повторную продажу продукта при помощи кнопки «Продажа».




Под блоком расположена таблица товаров, которые были перемещены. В таблице присутствует код товара, нажав на который можно перейти его к карточке. Более подробно о продуктах можно почитать в разделе «Продукты» или по этой ссылке .


Колонка «Действия» содержит три ссылки:
«История перемещений данного продукта» - позволяет просмотреть все перемещения конкретного продукта с момента его оприходования на складе.




«Редактировать» - позволяет внести некоторые правки в товар. Изменения при этом будут внесены во все записи о продукте, начиная с оприходования.




«Удалить» - удалить запись о перемещении продукта в журнале, затрагивает только журнал перемещений.




Блок документов — служит для добавления документов. Документ добавляется в контекст заказа или проекта, в рамках которого происходило перемещение товара. О том, как работать с документами можно прочесть в разделе «Проекты» → «Просмотр информации о проекте и редактирование проекта» → «Управление документами проекта» или по этой ссылке .
В контексте заказов - «Заказы» → «Просмотр и редактирование заказа» → «Управление документами заказа» или по этой ссылке .
