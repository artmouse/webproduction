Просмотр финансов

CRM система OneBox обладает модульной структурой. Это значит, что функционал системы может расширяться при помощи специальных модулей. Такой подход позволяет гибко настраивать систему в целом, так как, в зависимости от набора подключенных модулей, она может быть сконфигурирована по-разному.


Пример модульной структуры можно увидеть в карточке проекта. Горизонтальное меню в карточке состоит из множества вкладок. Некоторые вкладки по умолчанию недоступны, а становятся доступны лишь только после подключения конкретных модулей. К таким вкладкам относится вкладка «Финансы».


За отображение финансовых операций в проекте отвечает модуль «Finance». Если данный модуль подключен к системе, то вкладка «Финансы» в карточке проекта будет выглядеть следующим образом:




Если в проект включены продукты, то по ним обязательно считается общая сумма, оплата и баланс. Финансы позволяют отследить эти показатели, а также отображают отдельные платежи в контексте проекта.


Страница финансов выглядит следующим образом:




Информация по балансу объединена в блоке «Платежи».
Если проект не содержит продуктов или платежей, то баланс платежей будет равняться нулю, как на вышеуказанном скриншоте.
При добавлении продуктов в проект баланс изменяется на отрицательный — это значит, что клиент должен оплатить магазину, заказанные товары.


Под блоком «Платежи» находится таблица платежей. Здесь отображается дата проведения платежа, его направление, сумма и счет, на который был проведен платеж.


Некоторые поля таблицы представлены ссылками. Нажатие на номер или дату платежа ведет к форме редактирования платежа. Нажатие на поле «Аккаунт» - к списку всех платежей в системе. Имя создателя платежа — ссылка к карточке контакта.


Для добавления платежа нужно воспользоваться ссылкой «Создать платеж» или «Добавить платеж», если таблица платежей на странице отсутствует.




Форма для добавления платежа выглядит так:




В правом верхнем углу находится знак «Х», который служит для возврата на предыдущую страницу без сохранения изменений.


Поле «На сумму» - обязательно для заполнения. Также, для добавления платежа важно указать клиента в проекте, иначе система не будет знать, кому адресован данный платеж. Добавить клиента можно на вкладке «Информация». Более подробно о том, как это сделать, можно прочитать в разделе «Проекты» → «Просмотр информации о проекте и редактирование проекта» или по этой ссылке .


«Счет» - представлен выпадающим списком, который формируется из списка всех кошельков в финансах.
«Категория оплаты» - поле для группировки расходов, за отображение которых отвечает функционал модуля «Finance».
Для указания банковских реквизитов нажмите на ссылку «Банковские реквизиты». Ссылка открывает два новых поля: «Референс код» и «Банковские реквизиты».




Также форма дает возможность прикрепить файлы, для чего следует воспользоваться полем «Файл».


После внесения всех данных, для подтверждения изменений нажмите на кнопку «Добавить платеж», которая находится внизу формы.
«Отмена» - возврат на предыдущую страницу без сохранения изменений.


Кроме этого, на вкладке платежей проекта находится блок «Ожидаемые платежи». Эта информация полезна для планирования. По таким платежам предусмотрено формирование отчетов, за что отвечает пункт «Отчеты», который находится в OneBox меню в левой части экрана.


Для добавления ожидаемого платежа нажмите на «Добавить ожидаемый платеж».




Список ожидаемых платежей формируется здесь же, в правой части экрана. Для удаления платежа нажмите на кнопку «х» напротив нужного платежа:




Страница добавления ожидаемого платежа выглядит следующим образом:




В правом верхнем углу находится знак «Х», который служит для возврата на предыдущую страницу без сохранения изменений.


Поля «Сумма» и «Дата платежа» обязательны для заполнения, без их указания система не сможет добавить платеж в базу и выдаст сообщение об ошибке.


После внесения всех данных, для подтверждения изменений, нажмите на кнопку «Добавить ожидаемый платеж», которая находится внизу формы.
«Отмена» - возврат на предыдущую страницу без сохранения изменений.
