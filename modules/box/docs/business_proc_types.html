Типы бизнес-процессов

CRM система OneBox дает возможность создавать собственные типы бизнес-процессов. Это полезно в том случае, когда стандартные типы процессов не вписываются в формат работы предприятия или же их нужно расширить.
Например, туристическая компания вряд ли будет использовать такие бизнес-процессы, как "заказы" или "проекты". Такой компании больше подойдет процесс, который будет называться «Туры». Данный функционал позволяет создать такой процесс.


Следующим шагом после добавления типа должна быть настройка структуры самого бизнес-процесса, а именно этапов, из которых он будет строиться. О том, как это происходит, можно прочесть в разделе «Бизнес-процессы» → «Создание бизнес-процесса» или по этой ссылке .


Новые типы могут строиться по тем же бизнес-процессам, что и заказы, задачи или проекты. Важно! Все процессы в системе имеют структуру бизнес-процессов. Для комфортной и гибкой настройки CRM системы OneBox понимание работы бизнес-процессов обязательно. Прочесть об этом функционале можно в разделе «Бизнес-процессы» или по этой ссылке .


Для того, чтобы создать собственный тип бизнес-процесса нужно активировать OneBox меню при помощи указателя мышки, затем нажать на пункт «Настройки».

Страница всех типов бизнес-процессов выглядит следующим образом:




Процессы объединены в таблицу, которая позволяет сортировать данные. Происходит сортировка по нажатию на заголовок колонки. Чтобы выполнить сортировку в обратном порядке, нужно повторно нажать на заголовок.
Под таблицей находится функционал для экспорта документа в форматы CSV, XLS, XML. Это реализовано с помощью меню, которое активируется по клику мышкой.


При наведении курсора на заголовок списка изображение курсора изменится на знак «?». Это означает, что в этот момент пользователь может перейти к настройкам отображения таблицы - для этого нажмите правую кнопку мышки.


Страница настроек состоит из нескольких блоков и выглядит следующим образом:




В правом верхнем углу находится знак «Х», который служит для возврата на предыдущую страницу без сохранения изменений.
Первый блок позволяется регулировать отображение колонок. Он представлен множеством checkbox`ов, каждый из которых отвечает за свою колонку в таблице. Если checkbox отмечен, значит колонка отображается, если нет — колонка в таблице отображаться не будет. Важно! Это работает не для всех колонок, некоторые ключевые колонки должны быть видимыми всегда: их checkbox`ы не реагируют на клики.


Следующий блок представлен полями:
«Показывать записей на странице» - отображение количества записей на странице. Максимально допустимое значение - 100 записей.
«Сортировка по умолчанию» - выбор колонки, по которой будет происходить сортировка, а также возможность сортировать по порядку или в обратном алфавитном порядке.


Внизу расположены кнопки, которые позволяют завершить работу с формой:
«Сохранить» - нажмите, чтобы изменения вступили в силу.
«Отмена» - возврат на предыдущую страницу без сохранения изменений.

Добавление типа

Форма для добавления нового типа выглядит так:




«Название» - название для нового типа. Задается в единственном числе, например, «Тур». Поле является обязательным для заполнения. Если оставить его пустым, то система выдаст сообщение об ошибке и не сможет завершить создание типа.
Название будет отображено в меню OneBox. Пролистайте меню при помощи колесика мышки или вертикальный полосы прокрутки:




Также в панели быстрого доступа:




«Тип» - поле задает тип процесса для системы и является обязательным. Изначально в системе существуют такие типы бизнес-процессов: project, issue, order — эти наименования уже зарезервированы, поэтому при создании нового типа название должно быть уникальным, иначе это приведет к ошибке.


«Страница добавления как для» - позволяет выбрать форму для добавления. Выпадающий список содержит три формы: для проекта, заказа и задачи.


«Id контента» - поле используется для связи типа и кастомной формы. Предназначено для разработчиков. В случае, если форма для добавления контента должна быть кастомизирована и подключена к новому типу, следует обратиться в техподдержку. О том, как это сделать можно прочесть в разделе «OneBox меню» → «Написать в техподдержку» или по этой ссылке .


После того, как все поля формы были заполнены, нажмите на кнопку «Вставить новую запись».

Редактирование типа

Форма для редактирования обладает аналогичным набором полей, что и форма для добавления типа. Также в ней присутствует два дополнительных поля.




«Множественное название» - нужно использовать, если название для нового типа имеет множественное число, например, «Туры». Если такое название задано, оно также будет отображено в меню OneBox:




После того как изменения были внесены, нажмите на кнопку «Редактировать запись», это приведет к сохранению изменений.
Для удаления типа воспользуйтесь кнопкой «Удалить запись».
Для перехода обратно к таблице всех типов воспользуйтесь вкладкой «Типы Бизнес-процессов», которая находится в верхней части страницы.
